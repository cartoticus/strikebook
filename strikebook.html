OWNER'S MANUAL
Strikebook SOP · keep this app healthy

1 · What this is, where your data lives

Strikebook is a single self-contained file. Everything — sessions, history, analytics — is stored on this device, in this browser. Nothing is uploaded anywhere. That means privacy by default, and it means backups are your responsibility (see §4).

2 · Scoring: the daily drill

Pin rack mode (default): tap the pins left standing after each ball, then hit the big count button. Tap nothing = strike. This mode feeds all leave/spare analytics.
Keypad mode: faster, counts only — those frames score perfectly but can't appear in the practice rack or spare ledger.
F records a foul (scores zero, full rack respotted per rule). ↩ undoes the last ball. The announcer and share cards run themselves.

3 · Sessions: finish, don't reset

When the night's games are done, tap 🏁 Finish session. That permanently banks everything into History and returns to the start screen. Reset is the destructive one — it discards the current session without saving. Rule of thumb: Finish always, Reset only for do-overs.

4 · Backup SOP — do this after every tournament

Auto-backup (desktop Chrome/Edge only): on the start screen, tap the backup status pill and choose a folder — ideally your synced Google Drive/iCloud/Dropbox folder. From then on, every Finish session writes a dated backup file there automatically. This isn't available on iPhone or in most mobile browsers — the pill will say so plainly rather than pretend it works.

Manual backup (works everywhere): History → ⬇ Save backup file → store the JSON in a cloud folder you keep. To restore or move phones: open Strikebook on the new device → History → ⬆ Load backup. Imports merge safely — loading the same file twice never duplicates. Why this matters: clearing browser data (or iOS storage housekeeping) can erase on-device history without warning. A backup file — auto or manual — cannot be erased by a browser.

5 · How the analytics compute

Career stats group by a hidden bowler ID, so renaming a bowler never splits their history. Leave percentages count only pin-rack entries. A leave needs 4 attempts before it's ranked (under that, it's held in the "keep bowling" note). Splits are tracked separately so they don't drag down the makeable-spare percentage. The drill list estimates ~11 pins of scoring cost per missed spare — an approximation, flagged as such. Game Craft adds Clean Frame % (marks per frame, works even for keypad games) and Bounce-Back % (how often a mark follows an open). Spare Classes rolls every leave into three difficulty tiers — singles, makeable multi-pin, splits — and the Nemesis Pin is simply the single-pin leave you see most. The Ball-by-Ball block adds first-ball average (the strongest predictor of scoring), strike rate, double rate (strikes that follow strikes), clean-10th %, and a consistency figure (± your game-to-game swing). Game-Position Average compares your 1st vs 2nd vs 3rd game to reveal warm-up or fatigue patterns; it needs a few multi-game sessions before it appears. Lane condition (House / Sport / Challenge, plus an optional pattern name) can be set on the start screen or added later by editing any session's label — it's stored so future analytics can compare how a bowler scores across easy house shots vs tough sport patterns. You can pick a known pattern from the list or free-type any name, which matters because tournament patterns like Junior Gold are kept secret until after the block. When a pattern name is set, the session page shows a 🔍 Look up in the Kegel pattern library link that opens Kegel's official library (and copies the name so you can paste it into their search) — handy for pulling the actual pattern sheet before practice.

6 · Baker mode, drills & report cards

Baker mode: pick "Baker" as the session type on the start screen with 2+ bowlers. You'll score one shared sheet where bowlers alternate frames (lead-off throws frames 1, 6; second throws 2, 7; and so on). The app shows whose turn it is on every ball. Baker games don't produce individual game scores, but each frame's pin leaves are still credited to the bowler who threw it — so a kid's spare practice data keeps building even from Baker play.

Spare drill: on any bowler's page, tap 🎯 Start spare drill. The app deals target leaves weighted toward that bowler's weakest spares, you set them up (or shoot them on a full rack) and tap made/missed. Drill results are stored separately from match play, so they don't inflate real game stats.

Coach report card: tap 📋 Coach report card to generate a one-page image of a bowler's key stats — perfect for texting a parent or printing for a practice folder. It uses the phone's share sheet, or downloads if sharing isn't available.

7 · Sharing a bowler's data (handoff)

Score a kid on your phone, then hand off just their games so they land on the kid's own device — separate from the all-in backup. On any bowler's page, at the bottom, tap 📤 Send [name]'s data. That produces a small file with only that bowler's sessions; send it however you like (text, AirDrop, email). On their phone, they open History → ⬆ Load / import and pick the file. Strikebook detects it's a single-bowler file and asks whether to merge into their existing history or import as new — always a confirmation, never automatic. Sessions already on their phone are skipped, so sending the same file twice never duplicates anything. After the first merge, both devices share that bowler's hidden ID, so future handoffs line up automatically.

Remove from this device: the red button beside it deletes only that bowler's data from your phone — useful once you've handed a kid off and don't need to carry them anymore. It never touches their copy. If the bowler shared a session with others, only their part is removed and the rest stays.

Backup vs handoff — the difference: Save backup file is everyone at once, for protecting your data. Send [name]'s data is one person, for giving it to them. Load / import accepts either kind.

8 · Maintaining the app file

Keep one master copy of strikebook.html in your cloud drive. Your data is keyed to where the file lives — browser storage belongs to the page's location. So: replacing the file at the same location with a newer version keeps all history; moving it (new folder, new host, new domain) orphans the old data. Before any move: export a backup, move, then import. To modify the app, give the current file to Claude and describe the change; test with a throwaway session before trusting a new version at a tournament.

9 · Installing on a phone (iPhone & Android)

Strikebook lives at a web address, so there's no app store and no download — you "install" it by saving the page to your home screen. It then opens fullscreen with its own icon, works offline, and stores history right on the device. Send your team the Strikebook URL and these steps.

iPhone / iPad (must use Safari):
1. Open the Strikebook link in Safari — not Chrome, not the in-app browser inside Messages or Instagram. If it opened somewhere else, copy the link and paste it into Safari's address bar. This step matters most on iPhone: only Safari can add to the home screen, and home-screen apps get more durable storage than a regular tab.
2. Tap the Share button (the square with an up-arrow, at the bottom or top of Safari).
3. Scroll down the share list and tap Add to Home Screen.
4. Tap Add in the top corner. The bowling-pin icon appears on the home screen — open it from there from now on, not from Safari.

Android (Chrome):
1. Open the Strikebook link in Chrome.
2. Tap the ⋮ menu (three dots, top-right).
3. Tap Add to Home screen (on some phones it reads Install app).
4. Confirm with Add / Install. The icon lands on the home screen; some phones may pop up a one-time "install" banner you can just accept.

Why bother installing vs. just bookmarking: the home-screen version launches fullscreen with no browser bar, works with no signal (handy in an alley with bad wifi), and — on iPhone especially — its saved history is far less likely to be cleared by the browser's storage housekeeping. A plain Safari tab can lose data after a week of not being opened; a home-screen app doesn't.

First-launch note: because the app runs from a web address, the very first open needs a connection (after that it works offline). If nothing loads at an alley, it's almost always wifi — try again on cell data.

10 · Hosting & updates (GitHub Pages)

Strikebook is hosted at a GitHub Pages URL — that's the address you send people to install (§9).

To publish an update: in your GitHub repo, upload the new index.html and click "Commit changes" — same filename, same repo, so it replaces the old version at the same URL. That's what keeps everyone's on-device history intact (see §8): the URL never changes, only the file behind it does. Pages usually rebuilds within a minute; hard-refresh the phone to see the new version. Never rename the repo or change the URL once the team is using it — that's the "moving the file" scenario in §8, and it orphans history until everyone re-imports a backup.

11 · Troubleshooting

Share button does nothing: some in-app browsers block the share sheet — the card downloads instead; share the PNG manually. History looks empty: you may have opened the file from a different location — import your latest backup. Buttons unresponsive after an update: fully close and reopen the browser tab. When in doubt: export a backup first, then investigate.

12 · Sending feedback

On the start screen, 💬 Send feedback or request a feature opens a short form: pick the area it's about (scoring, analytics, Baker mode, and so on — or Feature request), mark it as a bug, idea, praise, or question, and write your message. Leaving the include app version & device info box ticked attaches the version, phone type, browser, and whether it's running as a home-screen app — that context solves most bug reports without a round of questions. Untick it if you'd rather not send it.

Where it goes: tapping Send opens your mail app with everything already filled in and addressed to the coach — you just hit send. If that doesn't work on your device — some phones have no mail app set up — tap 📋 Copy instead and paste it into a text message. Copy always works, including with no signal.

For the coach (setup): the destination is two lines near the top of the app file — FEEDBACK_EMAIL (currently set) and FEEDBACK_FORM. Swap in a Google Form URL in the second if you'd rather collect responses in a spreadsheet; the form takes priority if both are filled. If both are ever emptied, the Send button hides and only Copy is offered, so nothing breaks.

13 · Version history

The version you're running shows at the bottom of the start screen. To confirm everyone's current, have them check that number against the top of this list. After an update, the first launch shows a What's New screen listing only what changed since the version that device was on — so nobody has to read the whole history to catch up. A brand-new install skips it.

Older layout and reliability fixes were folded into 1.0.0 and aren't listed separately.
